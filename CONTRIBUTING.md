# Как внести изменения в проект

## Ветка dev

Все изменения вносятся через ветку `dev`.

## Создание Pull Request

1. Создайте ветку от `dev`: `git checkout -b feature/your-feature`
2. Внесите изменения
3. Запушьте: `git push origin feature/your-feature`
4. Создайте Pull Request на ветку `dev`
5. Дождитесь прохождения CI проверок
6. После аппрува — мердж

## Требования к коду

- Код должен проходить линтер (ruff)
- Все тесты должны проходить
- Docker образ должен собираться без ошибок
